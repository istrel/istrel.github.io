Как внедрить?

Инструменты

  • Webdriver.IO - для запуска браузера в Selenium Webdriver
  • Mocha - для организации тестов
  • Sinon.JS - для создания стабов (фальшивых реализаций методов)
  • Chai.JS - для проверок утверждений
  • Karma runner - запуск юнит тестов
  • Rosie - для создания фабрик

Пререквизиты

  • ExpressJS для статики
  • Fixtures
  • Stubs middleware

Зачем оно надо?

Мотивация

  • Работающие тесты (в сравнении с кодом после тестов)
  • Как следствие, уверенность в изменениях и при рефакторинге
  • Возможность разработки кода до интеграции

Если тест не починился после провала?

  1. Заставить тест проходить без изменения кода
  2. Убедиться в работоспособности тестов
    • Откатить изменения и убедиться, что код работает
    • Переписать код заново

Если неизвестно, как написать первый тест?

  1. Написать код без тестов
  2. Написать проходящие тесты к коду
  3. Убедиться в работоспособности тестов
  4. Получить опыт
  5. Все переписать!

Предупреждение!

  • Инициализация - долго. Изменение - быстро.
  • TDD дополняет, но не заменяет ручное тестирование
  • Сохраняйте разумность и помните о техническом долге
